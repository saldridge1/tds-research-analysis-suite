# TDS Research Analysis Suite

**A practitioner-built UX research tool by Team Design Studios**

---

## Why This Exists

Good design begins before a single wireframe is drawn. It begins in the field — in the careful observation of real people, in the patterns that emerge from honest data, and in the discipline of turning raw findings into actionable insight.

This tool was built out of that conviction.

The TDS Research Analysis Suite is a browser-based workspace designed for UX researchers and designers who take methodology seriously. It supports five core research activities — from participant card sorting to session transcription — within a single, cohesive interface. No subscriptions. No enterprise walls. Built by a practitioner, for practice.

---

## Modules

### Card Sorting
An open sort environment for uncovering participant mental models. Cards are read in full before sorting, then prioritized into categories that reflect natural groupings. Supports custom cards and custom categories. Ideal for navigation architecture, information hierarchy, and content taxonomy research.

### Heat Map
Click-tracking visualization with two modes: a built-in wireframe mockup, or a live URL loaded directly into the canvas. Interaction points accumulate intensity over time, revealing attention patterns and navigation behavior. Designed for usability testing and interaction analysis.

### Field Guide
A structured, editable observation table that adapts its columns to the research method in use — User Interview, Usability Study, Contextual Inquiry, or Survey Analysis. Every row includes a dedicated Comments field so researcher notes stay anchored to the observation that generated them. Exports to a dated CSV file for archiving and analysis.

### Affinity Diagram
A freeform board for clustering insights into themes. Notes are color-coded, draggable, and editable in place. Cluster labels can be added and repositioned to build the thematic architecture that bridges raw data and design decisions.

### User Flow Transcription
A dual-panel module combining a timestamped path timeline with an annotatable session transcript. Path elements and time intervals are logged as discrete steps. Audio recordings can be uploaded and transcribed, with each entry supporting inline annotations. Designed for researchers who process flows temporally rather than linearly.

## Design Philosophy

Clarity is the standard. Every module in this suite reflects a belief that research rigor and design quality are not in tension — they are the same discipline practiced at different scales.

This tool does not automate insight. It creates the conditions for it.

---

## Tech Stack

- Vanilla HTML, CSS, JavaScript — no frameworks, no dependencies
- Single-file architecture (`index.html`) — runs in any modern browser, no build step required
- Google Fonts: DM Serif Display + DM Sans
- Canvas API for heat map rendering
